「Webコンテンツ作成」の成果

本テーマについて

Webページの作り方の基礎を修得することが目的。Webコンテンツの基本的な記述をするためのHTMLや、Webページのレイアウトを指定するためのCSS、インタラクティブなページを作るためのJavaScriptなどについて学習する。

プロジェクト課題について

指定した都市の天気や最高気温、湿度や風速などを検索して表示するWebページを作った。検索ボタンをクリックするとラジオボタンで選択している都市の天候に関するデータの一覧が表示される。ラジオボタンのラベルの名前は空白を使って綺麗に2行で表示されるようにした。CSSでタイトルを中央に寄せることで見栄えが多少良くなるようにした。また、if文を使い天気によって条件分岐させ、検索した都市の天気に応じた天気マークも表示されるようにした。

プロジェクト課題で作成したページ: project.html

数当てゲーム

ランダムに生成された1〜10の整数を予想し当てるゲーム。ユーザが1〜10の整数を入力すると、その数字が当たっているかどうかを表示する。ユーザは3回まで予想することができ、予想が外れていた場合は正解の数字より大きいか小さいかを表示する。3回とも予想が外れた時は最後に正解を表示する。予想を当てるか、3回とも予想を外すとゲームが終了する。ゲームが終了しても数字を入力すると、既にゲームが終了しているというメッセージを表示する。第4回講義でDOMプログラミングを学び、予想の結果をWebコンテンツに直接表示できるようになった。さらに第5回講義でイベント処理を学び、実際に数を予想して結果が帰ってくるようになり、数当てゲームが完成した。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

WebページはHTML、CSS、JavaScript、画像・音声・動画の4つの部品に大きく分けられることがわかった。HTMLでよく使われる要素や、ハイパーリンクの設置のやり方、画像埋め込みのやり方など、HTMLを使ったWebページの作り方の基本を学んだ。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

HTMLで作られたページを装飾するCSSについて学んだ。装飾したい要素を指定するセレクタ、装飾の種類を指定するプロパティ名、装飾の具体的な値であるプロパティ値などのCSS書式を使ったWebページの装飾の仕方が分かった。

演習の成果: 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

Webページ上で複雑な機能を実現するプログラミング言語であるJavaScriptについて学んだ。JavaScriptにおける変数宣言や比較演算、メソッドの使い方などについて学んだ。また、デバッグを効率よく行うための開発者ツールの使い方が分かった。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

JavaScript などのプログラムによってWeb コンテンツの内容を読み取ったり，コンテンツに変更を加えたりする処理が書けるようになるDOMプログラミングについて学んだ。JavaScriptはコンソールに文章を表示する方法しか知らなかったが、これによりWebコンテンツに直接文章を表示できるようになった。

演習の成果: 演習4-2〜4-5: DOMプログラミング

第5回 イベント処理

ユーザーがマウスでクリックしたり、Webページのロードが完了した時などのイベントが発生した時にJavaScriptプログラムが動作するイベント処理について学んだ。これにより、ユーザーの操作に応じてWebコンテンツに動きを与えることができるようになった。

演習の成果: 演習5-3: イベントを選択する

工夫点，苦心点，強調点，感想

情報メディア実験のWebコンテンツ作成の授業を通して、Webページをどのようにして作っているかということを学ぶことができた。本講義は全体を通して難しかったが、Webページは普段からよく使う身近なものなので、Webページの作り方は学んでいて楽しかった。本講義では全体的に自分が思った通りにHTMLやCSSやJavaScriptを動作させることに苦戦していたことが多かった。これらはどれも使える構文の種類が非常に多いため、自分が想像したものを作るにはどの構文を使うべきか、ということに悩むことが多かった。また、構文の種類が多いためそれぞれの構文にどのような効果があるか、ということもなかなか覚えられず苦戦した。プロジェクト課題を作る際には、今まで学んだ内容をほぼ全て使うことになったので、今まで学んだ構文の内容を思い出すことにも苦戦した。このように自分の想像した通りにWebページを作ることに苦戦していたが、最終的に自分の思い通りのページが完成した時はとても嬉しかった。CSSを使ってWebページを装飾する時は、このようにつくってくださいというような指示が基本的に無く、自分で装飾を考えて作ることも難しかった。しかし、それと同時に自分で文字や背景の色づかいや文字の配置場所を考えることは楽しいとも感じた。このような自由度の高い課題の時には、なるべく見やすいページを作ることを心がけた。具体的には、装飾をつけすぎずになるべくシンプルな構造にして、文字や背景に色をつける際にも濃い色を広い面積に使うことは極力避け、目に優しくするようなイメージで色をつけることを心がけた。第5回講義でイベント処理を学んでからはユーザーに操作をしてもらうWebページを作り始めたので、その際はなるべく操作がしやすいようなページになるように工夫した。今回の授業が始まった時点では、Webコンテンツを作成する際に使うHTML、CSS、JavaScriptについて全く知らない状態だった。なので、第1回講義でHTMLの基本的な使い方を学ぶ段階でもわからないことが多く、常に苦戦しているような状態で授業についていくだけでも精一杯だった。しかし、Webコンテンツを作ることは楽しかった上に、問題が起きた時に原因を調べて解決するという一連の流れをよく行ったので、問題を解決する能力も得られたと思った。今後もWebコンテンツを作成する機会があったら、今回学んだことをよく活用し、それ以外の時も今回の講義を通して得た問題を解決する能力を活用していきたい。
